# TKT-003: Consolidate 23 CRUD table implementations into generic DataTable component

## Problem
There are 23 table implementations across the codebase that follow nearly identical patterns:
- deductions-table, earnings-table, benefits-table, pensions-table
- reimbursements-table, salary-sacrifice-table, orders-table, statutory-table
- pay-template-*-table (7 variants)
- bank-accounts-table, workpattern-table
- paygroup-employee-table, employee-table, payrun-table

Each contains:
- useState for create/update/selected state
- trpc mutations for delete/fetch
- useMemo for ColumnDef array
- DropdownMenu with Edit/Delete actions
- DataTable component wrapper

## Solution
Create a generic `<CrudTable>` component that accepts:
- Column definitions
- Entity type
- CRUD mutation hooks
- Optional action overrides

## Impact
~23 components reduced to configuration objects, significantly reducing code duplication.
