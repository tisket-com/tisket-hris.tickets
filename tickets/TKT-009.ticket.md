# TKT-009: Add data-slot attributes to UI components for consistency

## Problem
The tisket reference project uses `data-slot` attributes on all UI components:
```tsx
<element data-slot="button" className={...} />
<element data-slot="card" className={...} />
<element data-slot="badge" className={...} />
```

This pattern enables:
- CSS-in-JS identification
- Testing selectors
- Debugging

tisket-app components may not follow this convention.

## Solution
Add `data-slot` attributes to all `/components/ui/` components matching tisket's pattern.

## Reference
See tisket's `/components/ui/` for the pattern.
