# TKT-019: Employee refactor

## Overview
Refactor the employee page structure to improve navigation and reduce UI clutter.

## Part 1: Move employee route
- [x] Move `app/[orgId]/employee/[employeeId]` to `app/[orgId]/employees/[employeeId]`
- [x] Update link in `employee-table.tsx`
- [x] Update navigation paths in `lib/navigation.ts`

## Part 2: Replace sidenav with tabs
Replace the current `SecondarySidebar` navigation with a tab design similar to the workspace page in tisket/client (Projects/Teams tabs).

Tab groups:
- **Setup**: Overview, Basic Information, Employment, HMRC Details, Salary, Banking, Pay Template
- **Activity**: Notes, Leave, Task List, Timeline, Documents

## Part 3: DetailCard component
Create a reusable `DetailCard` component for displaying readonly data with a "Manage" button that opens a sheet for editing.

### Architecture proposal

```tsx
interface DetailCardProps {
  title: string
  description?: string
  children: React.ReactNode  // Readonly display content
  sheet?: React.ReactNode    // Sheet content for editing
  sheetTitle?: string
  sheetDescription?: string
}
```

Usage pattern:
```tsx
<DetailCard 
  title="Banking Details"
  description="Employee bank account information"
  sheetTitle="Edit Banking Details"
  sheet={<BankingForm employee={employee} />}
>
  <DetailCard.Field label="Account Name" value={banking.accountName} />
  <DetailCard.Field label="Sort Code" value={banking.sortCode} />
  <DetailCard.Field label="Account Number" value={banking.accountNumber} />
</DetailCard>
```

Benefits:
- Consistent pattern across all employee data sections
- Reduces form clutter - only shows forms when actively editing
- Separates read vs write concerns
- Can be reused for other entity detail pages
