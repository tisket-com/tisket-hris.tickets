# TKT-008: Adopt OKLCH color space from tisket reference project

## Problem
The tisket reference project uses OKLCH color space for perceptually uniform colors:
```css
--background: oklch(1 0 0)
--foreground: oklch(0.145 0 0)
--primary: oklch(0.205 0 0)
--destructive: oklch(0.577 0.245 27.325)
```

tisket-app may be using hex or HSL colors which don't provide the same perceptual consistency.

## Solution
Migrate CSS custom properties to OKLCH format matching tisket's approach:
- Light mode variables
- Dark mode overrides
- Sidebar-specific colors
- Chart colors

See `/Users/jim/Projects/tisket/client/src/styles/globals.css` for the color system.
